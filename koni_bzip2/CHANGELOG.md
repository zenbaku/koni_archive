# Changelog

## 0.10.0 (2026-07-18)

- Initial release: BZip2 **reading and writing** for the koni_archive ecosystem.
  - Writing (`Bzip2WriteFormat`) compresses one byte stream — `.bz2` is a
    single-member container — via the new `Bzip2Encoder` in `koni_codecs`
    (RLE1 → Burrows–Wheeler transform → MTF/RLE2 → length-limited Huffman).
    `blockSize100k: 1..9` selects the block size like `bzip2 -1`..`-9`
    (default 9). No encryption; a password is rejected. The output is
    byte-decodable by `bzip2` / libbz2, verified against the `bzip2` CLI and
    byte-identical across the VM, dart2js, and dart2wasm.
  - A bare `.bz2` opens as a single-entry archive; `.tar.bz2` / `.tbz2` /
    `.tbz` presents as the inner TAR via layered detection, through the new
    `Bzip2DecompressedByteSource`.
  - Decodes the bzip2 format (`BZh1`–`BZh9`) via `koni_codecs`, one block
    (≤ 900 KiB) at a time so memory stays bounded; concatenated streams are
    handled. Block and stream CRCs are always verified.
  - `.bz2` stores no filename, timestamp, or decompressed size, so the entry is
    named from the container and its `uncompressedSize` is `-1` (unknown —
    reading still yields the full content). The layered `.tar.bz2` decodes the
    whole container at open (no size metadata to probe), capped by
    `maxContainerDecodeSize` (falling back to `maxEntrySize`).
  - Typed errors: randomized blocks (a deprecated pre-0.9 bzip2 feature) and any
    failed CRC.
  - Fixtures generated by `bzip2` 1.0.8 (committed, provenance manifest);
    verified byte-for-byte on the VM, dart2js, and dart2wasm; fuzz smoke in CI.
