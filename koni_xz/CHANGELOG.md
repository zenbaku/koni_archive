# Changelog

## 0.10.0 (2026-07-18)

- Initial release: XZ **reading and writing** for the koni_archive ecosystem.
- **Writing** (`XzWriteFormat`): compresses one entry with LZMA2 as a single
  block with a CRC-64 check; the output is decodable by `xz` / liblzma. `.xz` is
  a single-member container, so the writer takes exactly one file entry (a
  second `add*`, a directory/link/other type, or a password is rejected — xz
  has no encryption); empty input writes a valid zero-block stream (byte-
  identical to `xz < /dev/null`). `.xz` stores no filename, so a write-then-read
  round trip preserves content, not the name. Single block + CRC-64 + no
  transform filters is the MVP (matches default `xz`); the payload is buffered
  while encoding (the one-shot `Lzma2Encoder`'s window). Verified by a
  write→read round trip on VM + dart2js + dart2wasm, with `xz -d` interop as a
  VM-only extra and the empty stream asserted byte-identical to `xz`'s.

### Reading

- A bare `.xz` opens as a single-entry archive (named from the container —
  `.xz` stores no filename or timestamp); `.tar.xz` / `.txz` presents as the
  inner TAR via layered detection, through the new `XzDecompressedByteSource`
  (block-by-block decode + in-memory cache).
- The `.xz` container is parsed from the end (stream footer → index → block
  table) across any number of concatenated streams, with a reconciliation check
  that rejects a walk that does not line up with the file length.
- Each block is decoded independently: LZMA2 (via `koni_codecs`) into a buffer
  sized from the index, then the transform filters (delta / x86 BCJ)
  reverse-applied, then the integrity check verified. Checks: None, CRC-32,
  CRC-64, and SHA-256 (`verifyChecksums: false` opt-out).
- Decompression-bomb guards: a block over `maxEntrySize` is rejected before
  allocation, and a layered `.tar.xz` honors `maxContainerDecodeSize` (falling
  back to `maxEntrySize`) at open.
- Typed errors (never a silent mis-read): non-x86 BCJ filters, a non-zero BCJ
  start offset, reserved check ids, and non-reconciling stream concatenation.
- Fixtures generated by `xz` (committed, provenance manifest); verified
  byte-for-byte on the VM, dart2js, and dart2wasm; fuzz smoke in CI.
- Requires `Crc64` (CRC-64/XZ), new in `koni_archive_core` 0.10.0.
